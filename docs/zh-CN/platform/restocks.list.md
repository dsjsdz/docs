📦 补货日志管理
一、功能概述

补货日志用于记录售卖机每一次补货操作的详细信息，包括补货时间、补货员、补货商品及数量等。
该模块帮助商户监控库存变化、分析补货效率、定位库存异常，并为库存管理与出货统计提供数据支持。

二、日志来源与更新机制

补货日志由系统或人工操作生成：

商户或补货员在后台或移动端提交补货操作

系统记录操作时间、补货人、售卖机编号、货道编号、补货数量

补货数据与库存同步，保证库存数据实时更新

所有补货记录实时同步至云平台，便于追踪和审计

三、主要功能

1. 📋 补货记录列表

按时间倒序显示所有补货操作

支持筛选条件：

补货时间范围

补货人姓名

售卖机编号 / 名称

商品名称

货道编号

支持关键字搜索快速定位记录

2. 🔍 补货详情

点击任意记录可查看完整补货信息：

补货编号 / 日志编号

操作人 / 补货员姓名

所属商户 / 售卖机名称

货道编号与商品信息

补货数量与剩余库存

操作时间

备注信息（如损坏商品、特殊情况说明）

3. ⚠️ 异常补货识别

系统可标注异常补货事件，如：

补货数量超过货道容量

库存不足

重复补货

异常记录会提醒商户或管理员进行核查

4. 📈 补货统计与分析

按售卖机、货道或商品维度统计补货次数与数量

可分析补货效率、库存消耗趋势

支持导出报表（Excel / CSV）用于库存管理或财务审计

5. 🔄 补货操作权限

支持后台操作和移动端扫码补货

补货操作会生成日志，确保可追溯性

四、权限与角色
角色 权限说明
商户管理员 查看、添加、编辑补货记录，审批异常补货
补货员 提交补货记录，查看自己操作的记录
区域管理员 查看负责区域补货日志，不可操作
技术支持 可查看补货记录和库存状态，便于设备维护
财务人员 查看补货数据和数量统计，用于成本核算
五、使用示例

例：商户“张三便利”补货“学校A区售卖机”

补货员登录后台或移动端 → 补货记录 → 选择售卖机

选择货道 A01 → 补货 50 件“巧克力棒”

系统生成补货日志：

补货人：李四

时间：2025-10-25 09:15

商品：巧克力棒

数量：50 件

货道 A01 原库存：20 → 补货后库存：70

商户可在后台查看补货统计和库存趋势，优化下次补货计划
