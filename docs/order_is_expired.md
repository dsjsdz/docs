# 订单20分钟未支付

平台将订单状态更新(失效)，同时将原订单的下单数量返回给设备库存(需要商户自己主动更新数据)。

平台会推送事件到 `notify_url` 可访问的公网URL地址，请求方式 `post`，接收出货结果。

:::tip
注意: 因接口返回的字段使用 ``protobuf``, 部分字段值为空 或 false 时，默认不传
:::

| 参数     | 类型 | 说明   | 必传 |
| -------- | ---- | ------ | ---- |
| order_id | int  | 订单id | ✓    |

## 请求结果(失败)

根据返回的 [错误代码](error_code.md) 进行排查:

```json
{
  "code": 60002,
  "message": "商品已下架",
  "data": {}
}
```
